Flagship project

OrderFlow — Business Process & Ticketing System

A production-shaped serverless application on AWS.

A ticketing and workflow system for a warehouse and delivery operation. Tickets move through five distinct roles — creator, stock controller, driver, buyer and admin — with drivers submitting proof-of-delivery photos from the field. Every action is written to an audit trail.

What makes it interesting

  • Infrastructure as code. The whole AWS stack is defined in TypeScript with AWS CDK — nothing was clicked together by hand, and dev and production are the same definition with a different flag.
  • The database was designed around its access patterns. DynamoDB single-table design with four secondary indexes, chosen up front from how the app actually queries — not bolted on afterwards.
  • Real authorization. A Cognito-authenticated API with a shared permissions layer that resolves a user's role from three possible sources and fails closed.
  • Tested and continuously integrated. 54 automated tests covering permissions and the ticket workflow, run on every push by GitHub Actions alongside a type check and a full infrastructure build.

Xelvo — Real-Time Private Messenger

One codebase, shipped as a web app and an Android app.

A one-to-one messaging app. Sign in by phone number, add a contact by QR code or invite link, then chat in real time — with typing indicators, read receipts, reactions, replies, editing, unsend, media and location sharing.

What makes it interesting

  • Security is enforced by the database, not the app. 54 row-level-security policies in PostgreSQL make it structurally impossible to read a conversation you aren't part of — even if the frontend has a bug.
  • Data integrity designed in. Database constraints guarantee a conversation between two people can only ever exist once, no matter who starts it.
  • Web and native from one codebase. Ships as an installable progressive web app and as a native Android build, with push notifications on both.
  • Honest about its limits. It isn't end-to-end encrypted, and the privacy screen says so in plain language rather than implying a guarantee it can't make.

Snake World — Competitive Arcade Game

Global leaderboards, and a server that assumes players will cheat.

A neon take on Snake with 40 level combinations, monthly competitive seasons, unique gamertags, daily challenges and worldwide leaderboards. Built in Godot and distributed as a Windows executable.

What makes it interesting

  • The leaderboard is server-authoritative. The game client cannot write to the score table at all — submissions go through a stored procedure that validates them, so a modified client has nothing to exploit.
  • Scores are checked for plausibility. Each level has a computed maximum realistic score based on its own difficulty; anything beyond it is rejected rather than ranked.
  • Country and season come from the server, never from the client, so neither can be faked. Rate limiting is applied per device and per IP, and IP addresses are stored only as hashes.

Couples Budget — Shared Finance Tracker

Two people, one budget, and a database layer I'm quietly proud of.

A web app for two partners to manage shared money: paired accounts, income and expense tracking, category budgets with budget-versus-actual reporting, recurring subscriptions that post themselves when due, and export to Excel or PDF.

What makes it interesting

  • It runs on two different databases from the same code. SQLite on a laptop, PostgreSQL in the cloud — a thin connection layer translates the query syntax, so no application code knows or cares which one it's talking to.
  • Credentials handled properly. Passwords are hashed with bcrypt and salted per user, and every query is parameterized, so SQL injection isn't possible.
  • Sessions time out after 30 minutes idle, and every administrative action is written to an audit log.

Deepclaim — Deep-Sea Mining Game

No art files, no sound files. All of it is code.

Drill down through a destructible seabed, fill a limited cargo hold, and surface to sell and upgrade before your oxygen runs out. Claiming a world's core wipes your run and pays out a permanent currency — twelve seas, thirty-six ores, and a long-term progression loop.

What makes it interesting

  • Every tile, sprite and sound is generated in code. The game ships with zero image or audio assets — all of it is produced procedurally at runtime.
  • Saves survive updates. The save file is versioned and merged onto current defaults when loaded, so a save written by an older build still works after new features land — and a corrupted file falls back to a fresh start instead of crashing.
  • It has a real test suite. 172 assertions that launch the actual game and drive it headlessly, catching what tests on isolated functions would miss.
